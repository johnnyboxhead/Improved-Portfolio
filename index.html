My Portfolio

River-Rat

This was a group project to provide key stats on the James River in Richmond, VA that aren't easily accessable/undestandable elsewhere. This is a full MERN app using MongoDB to story user data/comments, Passport for authentication, and hitting both the USGS and Google Maps APIs.


Key elements in this project were the ability to obtain river data via API, locate parks, obtain key information about each park, create users (including username/password), ability to retrieve user info, ability to create comments, and the ability to retrieve comments. My parts of the project were: connecting to the USGS API, syling the API data retrieved, creating the logic around activity recommendations,creating the route and logic for submitting comments, creating the outfitters page that displays a list of merchants, writing logic to ensure unacthenticated users couldn't get to the authenticated space, and logic around the component updates throughout the app.

SpeakIT

This was a group project with a concept of connecting users who want to learn how to speak other languages with users who are already fluent speakers of those other languages. This would be a way to learn another language naturally/conversationally without using a program like Babel or Rosetta Stone to do so.


Key elements in this project were the ability to create users (including username/password), ability to retrieve data from a database, ability to connect users, and the ability to chat. My parts of the project were: writing the routes for returning information from our users table, developing the routes/models for the connections table, and returning connected users from the connections table.

GatheRVA

This was another group project where we decided to tackle the challenge of figuring out what to do next weekend in Richmond, VA. While there are plenty of event calendars, none of them aggregate inputs from both open source APIs and events created by users of the website, GatheRVA does just that.


Key elements in this project were the ability to pull data from APIs, parsing and rendering the API data on our webpage, and using Firebase storage to allow users to create new events. My parts of this project were pulling data from the eventful API, parsing it, then rendering it to the webpage. I also shot, edited, and added the video (including drone footage) to the background.

Bamazon

This was an individual project to create a Node application that simulates a store in Terminal/Gitbash. The users are able to make purchases which are then deducted from the store's inventory. Managers are able to add items and update inventory.


Key elements in this project were the ability to save the inventory in a MySQL database, querying the database to retrieve items, using Inquirer for users to select products, then using queries to update inventory. While this isn't much to look at, it was fun to create and to use.

Giftastic

This was an individual project to create a website that is able to retrieve and watch .gifs based on a user's input. The user can select which of the three pre-selected vehicles they want to see burn rubber or can add a vehicle of their choice. Once a vehicle button has been added to the top of the page the user can press that button to return 10 gifs of that vehicle. The user can then click on any thumbnail to start the .gif and click it again to stop it.


Key elements in this project were the ability to take input from a user and pass that into an call to the Giphy API. Once the return from the API is recieved it then has to be parsed and rendered on the front end.
